HTML Uniform Resource Locators

URL merupakan kata lain dari web address (alamat web).

URL bisa berupa nama (www.google.com), atau sebuah Internet Protocol (IP) address (192.68.20.50).

Kebanyakan orang memasukan nama ketika surfing (menelusuri internet) karena nama lebih mudah untuk diingat daripada nomor (IP).

URL - Uniform Resource Locator

Web browser meminta halaman dari web server dengan menggunakan sebuah URL.

Ketika kamu mengklik pada sebuah link pada halaman HTML, yang mendasari tag <a> bermaksud untuk sebah alamant pada web.

Uniform Resource Locator (URL) digunakan untuk alamat dokumen (atau data lainnya) pada web.

Sebuah web address, seperti http://www.w3schools.com/html/default.asp mengikuti aturan sintaksis:

Sintaksis URL:

scheme://prefix.domain:port/path/filename

Keterangan:

  • scheme - mendefinisikan tipe dari internet service (yang paling umum adalah http)
  • prefix - mendefinisikan sebuah domain prefix (default untuk http adalah www)
  • domain - mendefinisikan internet domain name (w3schools.com.com)
  • port - mendefinisikan sebuah port number atau host (default untuk http adalah 80)
  • path - mendefinisikan path pada server (If omitted: the root directory of the site)
  • filename - mendefinisikan nama dari sebuah dokumen atau resource (sumber)
Password yang dilindungi site mungkin menggunakan https dan/atau username dan password seperti:
https://username:password@www.w3schools.com/html/default.asp

Common URL Schemes

Tabel dibawah ini adalah daftar beberapa schemes yang umum digunakan:

Scheme Singkatan dari digunakan untuk
http HyperText Transfer Protocol web pages umum. Tidak dienkripsi
https Secure HyperText Transfer Protocol web pages terkunci dan aman
ftp File Transfer Protocol Downloading (pengunduhan) atau uploading (pengungahan) file
file File pada komputer kamu

URL Encoding

URL juga bisa dikirim lebih ke Internet menggunakan ASCII character-set. Jika sebuah URL berisikan karakter diluar ASCII set, URL tidak harus dikonversikan (ditafsirkan).

URL encoding mengkonversikan karakter non-ASCII kedalam sebuah formst yang bisa ditransmisikan (dikirimkan) lebih ke internet.

URL encoding menggantikan karakter non-ASCII dengan sebuah simbol "%" diikuti dengan hexadecimal digit.

URL tidak bisa berisi spasi. URL encoding secara normal mengganti spasi dengan tanda plus (+) atau %20.

Try It Yourself

Jika kamu mengklik "Submit", browser akan URL encode inout sebelum ia dikirim menuju server.

Halaman pada server akan menampilkan input yang diterima.

Coba beberapa input (masukan) lain dan klik Submit lagi.
