Pengenalan HTML5

Apa yang Baru dalam HTML5 ?

Pendeklarasian DOCTYPE untuk HTML5 sangat sederhana:

<!DOCTYPE html>

pendeklarasian character encoding (charset) juga sangat sederhana:

<meta charset="UTF-8">

Contoh HTML5

<!DOCTYPE html>

<html>
<head>

<title> Judul Halaman </title>
<meta charset="UTF-8">

</head>
<body>

Beberapa konten HTML..

</body>
</html>
Default character encoding dalam HTML5 adalah UTF-8.

New HTML5 Elements

Element baru yang paling diminati adalah:

Element semantik baru seperti: <header>, <footer>, <article>, dan <section>.

Form control attribute baru seperti number, date, time, calendar, dan range.

Element graphic baru: <svg> dan <canvas>.

Element multimedia baru: <audio> dan <video>.

Pada chapter HTML5 Support, kamu akan mempelajari bagaimana untuk "mengajarkan" browser lama untuk meng-handle HTML5 semantic.

New HTML5 API's (Application Programming Interfaces)

API's baru yang paling diminati adalah:

  • HTML Geolocation
  • HTML Drag and Drop
  • HTML Local Storage
  • HTML Application Cache
  • HTML Web Workers
  • HTML SSE
Local storage is a powerful replacement for cookies.
Penyimpanan lokal merupakan sebuah pengganti yang kuat untuk cookies.

HTML History

Sejak hari - hari yang lalu tentang web, telah ada banyak versi - versi HTML:

Versi Tahun
Tim Berners-Lee invented www 1989
Tim Berners-Lee invented HTML 1991
Dave Raggett drafted HTML+ 1993
Dave Raggett drafted HTML+ 1993
HTML Working Group defined HTML 2.0 1995
W3C Recommended HTML 3.2 1997
W3C Recommended HTML 4.01 1999
W3C Recommended XHTML 1.0 2000
HTML5 WHATWG First Public Draft 2008
HTML5 WHATWG Living Standard 2012
HTML5 W3C Final Recommendation 2014

Tim Berners-Lee invented the "World Wide Web" pada tahun 1989, dan Internet diberangkatan pada tahun 1990 an.

Dari tahun 1991 sampai 1998, HTML dikembangkan dari versi 1 sampai 4.

Di tahun 2000, World Wide Web Consortium (W3C) me-rekommendasi XHTML 1.0.Â 

Sintaksis (penulisan) XHTML merupakan sangat teliti (strict), dan para developers dipaksa untuk menulis dengan valid dan kode "well-formed" (formulir baik).

Di tahun 2004, WHATWG (Web Hypertext Application Technology Working Group) membentuk respon untuk slow W3C development, dan ketegasan W3C untuk menutuppengembangan HTML, pada dukungan tentang HTML.

WHATWG ingin mengembangkan HTML, konsisten dengan bagaimana web digunakan, selama menjadi cocok yang terbalik (being backward compalible) dengan versi - versi HTML yang dulu.

Pada periode 2004-2006, the WHATWG berinisiatif memperoloeh support oleh major browser vendor.

Pada 2006, W3C mengumumkan bahwa mereka akan mendukung WHATWG.

Pada 2008, WHATWA0G dan W3C diiputuskan dalam sebuah perpisahan:

WHATWG will develop HTML as a "Living Standard".

Standar kehidupan tidak pernah tercukupi, tapi selalu diperbaharui dan ditingkatkan. Features baru bisa ditambahakan, tapi fungsionalitas tidak bisa digantikan.

The WHATWG Living Standard dipublikasikan pada tahun 2012, dan terus - menerus diperbaharui.

W3C will develop a definitive HTML5 and XHTML5 standard, sebagai sebuah "snapshot" dari WHATWG.

The W3C HTML5 recommendation dirilis pada 28 oktober 2014.
