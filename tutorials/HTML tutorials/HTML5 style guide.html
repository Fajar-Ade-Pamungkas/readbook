HTML(5) Style Guide and Coding Conventions

Panduan Gaya dan Ketentuan Persandian HTML(5).

HTML Coding Conventions

Perjanjian Penyadian HTML

Para pengembang HTML sering kali ragu-ragu tentang coding style (kode tampilan) dan sintaksis untuk menggunakannya pada HTML.

Antara tahun 2000 dan 2010, banyak para pengembang web yang mengubah dari HTML menjadi XHTML.

Dengan XHTML, para pengembang dituntut untuk menulis secara valid dan "well-formed" code (kode format yang baik).

HTML5 merupakan sebuah bit lebih tidak rapih ketika ia sampai pada code validation.

Dengan HTML5, kamu harus menciptakan Best Practie, Style Guide, and Coding Conventions milik kamu sendiri.

Be Smart and Future Proof

Pintarlah dan buktikan masa depan

Konsekuensi menggunakan style, membuat ia mudah orang lain mengerti dan menggunakan HTML kamu.

Di masa yang akan datang, program seperti XML readers, mungkin mau membaca HTML kamu.

Menggunakan sebuah well-formed (formulir yang baik) "close to XHMTL" syntax, bisa menjadi pintar.

Selalu jaga your style smart (tampilan pintar kamu), tidy (rapih), clean (bersih) dan well-formed (formulir yang baik).

Use Correct Document Type

Selalu deklarasikan tipe dokumen sebagai baris pertama pada dokumen kamu:

<!DOCTYPE html>

Jika kamu ingin konsisten (berpendirian) dengan lowercase tag (penulisan dalam huruf kecil), kamu bisa gunakan:

<!doctype html>

Use Lower Case Element Names

Gunakanlah nama -nama element dengan huruf kecil

HTML5 mengizinkan pencampuran uppercase dan lowercase letters (huruf kecil dan besar) dalam nama element.

kita rekomendasikan penggunaan nama element lowercase:

  • Pencampuran nama dengan huruf besar dan kecil adalah buruk.
  • Para developer menggunakan nama huruf kecil (seperti dalam XHTML).
  • Lowercase terlihat bersih dan rapih
  • Lowercase lebih mudsh ditulis

Buruk

<SECTION>
<p> This is a paragraph. </p>
</SECTION>

Sangat Buruk

<SECTION>
<p> This is a paragraph. </p>
</section>

Bagus

<section>
<p> This is a paragraph. </p>
</section>

Close All HTML Elements

Tutuplah semua element HTML

Pada HTML5, kamu tidak hsrus menutup semua element (untuk contoh element <p>).

Kita rekomendasikan untuk selalu menutup semua element HTML:

Buruk

<section>
<p> This is a paragraph.
<p> This is a paragraph.
</section>

Bagus

<section>
<p> This is a paragraph. </p>
<p> This is a paragraph. </p>
</section>

Close Empty HTML Elements

Tutuplah Element HTML Kosong (Jomblo)

Pada HTML5, ini merupakan pilihan untuk menutup element empty (kosong).

Tidak Baik

<meta charset="utf-8">

Bagus

<meta charset="utf-8"/>

slash (/) diharuskan dalam XHTML dan XML.

Jika kamu mengira sofware XML tuk mengakses halaman kamu, ini mungkin ide yang bagus untuk selalu menggunakan slash untuk menutup element empty.

Use Lower Case Attribute Names

Gunakanlah nama atribute menggunakan huruf kecil.

HTML5 mengizinkan pencampuran huruf besar dan kecil dalam nama attribute.

Tapi kita rekomendasikan penggunaan nama attibute dengan lowercase:

  • Pencampuran nama dengan huruf besar dan kecil adalah buruk.
  • Para developer menggunakan nama huruf kecil (seperti dalam XHTML).
  • Lowercase terlihat bersih dan rapih
  • Lowercase lebih mudsh ditulis

Buruk

<div CLASS="menu">

Bagus

<div class="menu">

Quote Attribute Values

Berilah Tanda Kutip Pada Nilai Attribute.

HTML5 mengizinkan nilai attribute tanpa quotes (tanda kutip).

Namun, kita sarankan penggunaan tanda kutip untuk nilai attribute:

  • Kamu harus menggunakan quotes jika nilai berisikan spasi
  • pencampuran stayle tidak pernah baik
  • Nilai yang diberi tanda kutip lebih mudsh dibaca

Ini tidak akan bekerja, karena nilai berisi spasi:

Buruk

<div class=image flow>

Ini akan bekerja

Bagus

<div class="image flow">

Image Attributes

Attibute Gambar

Selalu gunakan attrihute alt pada gambar, Ini penting ketika gambar tidsk bisa ditampilkan.

<img src="html5.gif" alt="HTML5" style="width:120px; height:120px;">

Selalu definisikan ukursn gambar. Ini mengurangi flickering (kelap - kelip) karens browser bisa reserve spaspce untuk gambar sebelum mereka dimuat.

<img src="html5.gif" alt="HTML5" style="width:120px; height:120px;">

Spaces and Equal Signs

Tanda Sama dengan (=) dan Spasi

Spasi disekitar tanda sama dengan adalah legal (diperbolehkan):

<link rel = "stylesheet" href = "styles.css">

Tapi, spasi-less lebih mudah untuk dibaca, dan pengelompokan entitas lebih baik bersama:

<link rel="stylesheet" href="styles.css">

Avoid Long Code Lines

Hindarilah Baris Kode Terlalu Panjang

Ketikan menggunakan sebuah editor HTML, ini menyenangkan untuk menggeser layar kekanan dan kekiri untuk pembacaan kode HMTL.

Cobalah untuk menghindari panjang baris kode yang lebih dari 80 karakter.

Blank Lines and Indentation

Jeda baris dan indensasi

Jangan menjeda baris tanpa alasan.

Untuk kemampuan pembacaan, tambahakn blank lines untuk memisahkan besar atau blok kode logika.

Untuk kemampuan pambacaan, tambahkan 2 spasi indensasi. Jangan gunakan TAB.

Jangan gunakan blank line dan indensasi yang tidak diperlukan. Ini tidak diperlukan untuk menggunakan blank line antara short dan related items, Ini tidak memerlukan untuk mengidensasi setiap element.

Tidak perlu

<article>

<section>

<h2>Famous Cities</h2>

<div class="city">
<h2>London</h2>
<p>
London is the capital city of England. It is the most populous city in the United Kingdom, with a metropolitan area of over 13 million inhabitants.
</p>
</div>

</section>

</article>

Lebih Baik

<article>

<section>

<h2>Famous Cities</h2>

<div class="city">
<h2>London</h2>
<p>London is the capital city of England. It is the most populous city in the United Kingdom, with a metropolitan area of over 13 million inhabitants.</p>
</div>

</section>

</article>

Use Lower Case File Names

Gunakanlah Nama File Dengan Huruf Kecil

Kebanyakan web server (Apache, Unix) merupakan case sensitive tentang nama file:

london.jpg tidak bisa diakses sebagai London.jpg.

Web server lainnya (Microsoft, IIS) tidak merupakan case sensitve:

london.jpg bisa diakses sebagai London.jpg atau london.jpg.

Jika kamu menggunakan campuran huruf besar dan kecil, kamu harus sangat konsisten.

Jika kamu berpindah dari sebuah server case insensitive ke server case sensitive, bahkan error kecil pun akan mem-break (menghentikan) web kamu.

Untuk menghindari masalah ini, selalu gunakan nama file dengan huruf kecil (lower case) (jika memungkinkan).

File Extensions

File HTML harus memiliki extensi .html (atau .htm).

File CSS harus memiliki extensi .css

File JavaScript harus memiliki extensi .js

Differences Between .htm and .html

Perbedaan antara .htm dan .html

Tidak akan perbedaan antara ekstensi .htm dan .html. keduanya akan diperlakukan sebagai HTML web browser atau web server apapun.

Perbedaan merupakan kultural:

.htm "berbau" tentang sistem DOS awal dimana sistem dibatasi extensi menjadi 3 karakter.

.html "berbau" sistem operasi Unix yang tidak memiliki perbatasan ini.

Technically Differences

Perbedaan secara teknis.

Ketika sebuah URL tidak menentukan sebuah filename (seperti http://www.w3schools.com/css/), server mengembalikan sebuah default filename. Umumnya defsult filename adalah index.html, index.htm, default.html, default.html.

Jika server kamu dikonfigurasi hanya dengan "index.html" sebagai defaul filename, file kamu harus dinamai "index.html" bukan "index.htm".

Akan tetapi, server bisa dikonfigurasi dengan lebih daripada satu default filename, dan normalnya kamu bisa menyetel ulang sebanyak default filename yang dibutuhkan.

Akan demikian, full extension untuk file HTML adalah .html, dan tidak ada alasan .html tidak harus digunakan.
