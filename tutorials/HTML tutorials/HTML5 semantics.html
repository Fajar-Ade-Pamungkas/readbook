HTML5 Semantic Elements

Semantik adalah pelajaran tentang pengartian kata dan ungkapan atau ucapan (phrases) dalam bahasa.

Element semantik adalah element dengan sebuah arti.

Apa itu Element Semantik ?

Suatu element semantik dengan jelas mendeskripsikan atau menggambarkan arti element itu untuk browser dan developer.

Contoh element non-semantic: <div>, dan <span> - Tidak memberitahukan apapun tentang kontennya.

Contoh element semantic: <form>, <table>, dan <img> - Secara jelas mendefinisikan konten itu sendiri.

Browser Support

Element semantik HTML5 didukung di semua browser modern.

Pada penambahan, kamu bisa "mengajarkan" browser versi lebih lama bagaimana untuk menangani "unknown elements".

Bacalah lebih tentang ini di HTML5 Browser Support.

New Semantic Elements in HTML5

Banyak web site yang berisikan kode HTML seperti: <div id="nav">, <div id="header">, <div id="footer"> untuk mengindikasikan navigation, header dan footer.

HTML5 menawarkan element - element semantik baru untuk mendefinisikan bagian - bagian dari sebuah halaman web yang berbeda:

  • <header>
  • <nav>
  • <section>
  • <article>
  • <aside>
  • <footer>
  • <details>
  • <summary>
  • <figcaption>
  • <figure>
  • <main>
  • <mark>
  • <time>

HTML5 <section> Element

Element <section> mendefinisikan sebuah sesi (bagian) dalam sebuah dokumen.

Berdasarkan pada dokumentasi HTML5 W3C: "Section adalah sebuah pengelompokan tematik tentang konten, secara tipe dengan sebuah heading."

Sebuah Web site's home page bisa dirobek (split) kedalam sesi untuk introduction (pengenalan), content (isian), dan contact information (informasi kontak).

Input:

<!DOCTYPE html>
<html>
<head>
</head>

<body>

<section>
<h1>WWF</h1>
<p>The World Wide Fund for Nature (WWF) is an international organization working on issues regarding the conservation, research and restoration of the environment, formerly named the World Wildlife Fund. WWF was founded in 1961.</p>
</section>

<section> <h1>WWF's Panda symbol</h1> <p>The Panda has become the symbol of WWF. The well-known panda logo of WWF originated from a panda named Chi Chi that was transferred from the Beijing Zoo to the London Zoo in the same year of the establishment of WWF.</p>
</section>
</body>
</html>

WWF

The World Wide Fund for Nature (WWF) is an international organization working on issues regarding the conservation, research and restoration of the environment, formerly named the World Wildlife Fund. WWF was founded in 1961.

WWF's Panda symbol

The Panda has become the symbol of WWF. The well-known panda logo of WWF originated from a panda named Chi Chi that was transferred from the Beijing Zoo to the London Zoo in the same year of the establishment of WWF.

HTML5 <article> Element

Element <article> mendefinisikan kebebasan (independent), konten berisi dirinya sendiri (self-contained content).

Sebuah article harus membuat pengertian pada miliknya sendiri, dan article harus memungkinkan untuk membacanya secara bebas dari pengistirahatan website.

Contoh dimana element <article> bisa digunakan:

  • Forum post
  • Blog post
  • Newspaper article

Input:

<!DOCTYPE html>
<html>
<head>
</head>

<body>

<article>
  <h1>What Does WWF Do?</h1>
  <p>WWF's mission is to stop the degradation of our planet's natural environment,   and build a future in which humans live in harmony with nature.</p>
</article>
</body>
</html>
 

What Does WWF Do?

 

WWF's mission is to stop the degradation of our planet's natural environment,   and build a future in which humans live in harmony with nature.

Nesting Semantic Elements

Pada standar HTML5, element <article> mendefinisikan sebuah lengkapan, self-contained block of related elements.

Element <section> didefinisiksn sebagai element blok dari hubungan.

Bisakah kita menggunakan pendefinisian untuk memutuskan bagaiaman untuk menyarangkan (nest)? Atau tidak !

Di internet, kamu akan menemukan halaman HTML dengan element <section> berisikan element <article>, dan element <article> berisikan element <section>.

Kamu akan menemukan halaman dengan element <section> berisikan element <section> dan element <article> berisikan element <article>.

Newspaper: artikel olahraga dalam sesi olahraga, memiliki sesi teknis dalam setiap artikel.

HTML5 <header> Element

Element <header> menentukan sebuah header (kepala) untuk sebuah dokumen atau sesi.

Element <header> lebih baik digunakan sebagai container untuk konten pengantar.

Kamu bisa memiliki beberapa element <header> dalam satu dokumen.

Contoh berikut mendefinisikan sebuah header untuk sebuah article:

Input:

<!DOCTYPE html>
<html>
<head>
</head>

<body>

<article>
  <header>
    <h1>What Does WWF Do?</h1>
    <p>WWF's mission:</p>
  </header>
  <p>WWF's mission is to stop the degradation of our planet's natural environment,   and build a future in which humans live in harmony with nature.</p>
</article>
</body>
</html>
 
   

What Does WWF Do?

   

WWF's mission:

 
 

WWF's mission is to stop the degradation of our planet's natural environment,   and build a future in which humans live in harmony with nature.

HTML5 <footer> Element

Element <footer> menentukan sebuah footer (kaki) dari sebuah dokumen atau sesi.

Element <footer> lebih baik berisikan informasi tentang element isian itu sendiri.

Footer secara khas berisikan autor (penulis)dari dokumen, copyright information, links to terms of use, contact information dan sebagainya.

Kamu bisa memiliki beberapa element <footer> dalam satu dokumen.

Contoh:

<footer>  
<p>Posted by: Hege Refsnes</p>
  <p>Contact information: <a href="mailto:someone@example.com">   someone@example.com</a>.</p>
</footer>

HTML5 <nav> Element

Element <nav> mendefinisikan sebuah penyetelan dari navigation link.

Element <nav> diterangkan untuk large block of navigation links. Akan tetapi, tidak semua link dalam dokument harus berada didalam element <nav>!

Contoh:

<nav>
  <a href="/html/">HTML</a> |
  <a href="/css/">CSS</a> |
  <a href="/js/">JavaScript</a> |
  <a href="/jquery/">jQuery</a>
</nav>

HTML5 <aside> Element

Element <aside> mendefinisikan bebrapa konten disamping dari konten utama (seperti sebuah sidebar).

Konten aside lebih baik dihubungkan ke konten berlingkup (surrounding content).

Contoh:

<p>My family and I visited The Epcot center this summer.</p>
<aside>
  <h4>Epcot Center</h4>
  <p>The Epcot Center is a theme park in Disney World, Florida.</p>
</aside>

HTML5 <figure> and <figcaption> Elements

Dalam buku dan koran, ini merupakan umum untuk memilikk caption dengan gambar.

Tujuan dari sebuah caption adalah untuk menambahkan sebuah penjelasan visual ke sebuah gambar.

Dengan HTML5, gambar dan caption bisa dikelompokan bersa,a dslam sebuah element <figure>:

Contoh:

<figure>
  <img src="pic_mountain.jpg" alt="The Pulpit Rock" width="304" height="228">
  <figcaption>Fig1. - The Pulpit Rock, Norway.</figcaption>
</figure>

Element <img> mendefinisikan gambar, element <figcaption> mendefinisikan sebuah caption (keterangan atau judul).

Why Semantic HTML5 Elements?

Dengan HTML4, para pengembang menggunakan nama attribute favorit mereka untuk style page elements, seperti:

header, top, bottom, footer, menu, navigation, main, container, content, article, sidebar, topnav, ...

Ini membuat ia tidak mungkin untuk search engines (mesin pencari) untuk mengidentifikasi konten web page yang benar.

Dengan element HTML5 seperti: <header>, <footer>, <nav>, <section>, <article>, dengan ini akan menjadi lebih mudah.

Perdasarkan pada W3F, sebuah Web Semantik:

"Mengizinkan data untuk dibagikan dan digunakan kembali dari satu ke sisi lain applications, enterprises, dan communities."
