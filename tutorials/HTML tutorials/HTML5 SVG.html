HTML5 SVG

What is SVG?

Apa itu SVG ?

  • SVG berdiri untuk Scalable Vector Graphics (Grafik Sekala Vektor dapat dinaikan)
  • SVG digunakan untuk mendefinisikan grafik untuk Web
  • SVG adalah sebuah rekomendasi W3C

The HTML <svg> Element

Element <svg> (diperkenalkan di HTML5) adalah sebuah container (tempat) untik grafik SVG.

Canvas memiliki beberapa metode untuk penggambaran paths, boxes, circles, text, dan grafik gambar.

Differences Between SVG and Canvas

Perbedaan Antara SVG dan Canavas

SVG merupakan suatu bahasa untuk pendeskripsian grafik 2D (dua dimensi) dalam XML.

Canvas menggambar grapik 2D, on the fly (dengan sebuah JavaScript).

SVG didasari XML, yang mana artinya setiap elemen tersedia dengan SVG DOM, kamu bisa melampirkan JavaScript event handler untuk sebuah element.

Dalam SVG, setiap shape (bentuk) diingat sebagai sebuah object, Jika attribute dari sebuah object SVG diubah, browser bisa secara otomatis mengubah kembali shape.

Canvas diterjemahkan atau diubah pixel demi pixel. Dalam canvas, sekali grafik digambar, ia dilupakan oleh browser. Jika posisi canvas harus diubah, sekuruh layar perlu digambar ulang, termasuk object apa saja yang mungkin telah di-cover oleh grafik.

Comparison of Canvas and SVG

Perbandingan Canvas dan SVG

Tabel dibawah menampilkan beberapa perbedaan penting antsra Canvas dan SVG:

Canvas SVG
  • Resolution independent (bebas)
  • Tidak ada dukungan untuk event handler
  • Kemampuan menyumbang teks miskin (sedikit)
  • Kamu bisa menyimpan hasil gambar sebagau .png atau .jpg
  • Cocok untuk graphic-intensive games
  • Resolution independent (bebeas)
  • Mendukung Event handler
  • Sangat cocok untuk applikasi dengan area penyumbang luas atau besar (Google Maps)
  • Tidak cocok untuk aplikasi game.
