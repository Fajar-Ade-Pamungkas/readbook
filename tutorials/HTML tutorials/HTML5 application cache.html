HTML5 Application Cache

Dengan application cache, ini mudah untuk membuat versi offline tentang web application, dengan penciptaan sebuah cache manifest file.

What is Application Cache?

Apa itu Application Cache?

HTML5 memperkenalkan application cache, yang mana artinya bahwa sebuah web application di-cache dan dapat diakses tanpa koneksi internet.

Application cache memberikan sebuah aplikasi dengan 3 keuntungan:

  • Offline browsing - Para pengguna bisa menggunakan aplikasi ketika mereka sedang offline
  • Speed - cached resource (sumber - sumber yang di-cache) dimuat lebih cepat
  • Reduced server load - Browser hanya akan mengunduh perbaruan/perubahan sumber dari server.

HTML Cache Manifest Example

Contoh Manifest Cache HTML

Contoh dibawah ini menampilkan sebuah dokumen HTML dengan sebuah cache manifest (untuk offline browsing):

Input:

<!DOCTYPE html>
<html manifest="demo_html.appcache">
<head>
<title> Judul Halaman </title>
</head>
<body>

<p id="timePara"> <button onclick="getDateTime()">Get Date and Time</button></p>

<p><img src="img_logo.gif" width="336" height="69"></p>

<p>Try opening <a href="tryhtml5_html_manifest.htm" target="_blank">this page</a>, then go offline, and reload the page. The script and the image should still work.</p>

<script src="demo_time.js"></script>

</body>
</html>

Try opening this page, then go offline, and reload the page. The script and the image should still work.

Cache Manifest Basics

Untuk membolehkan application cahce, Masukanlah attribute manifest dalam tag <html> dokumen:

<!DOCTYPE HTML>
<html manifest="demo.appcache">
...
</html>

Setiap halaman dengan attribute manifest tertentu akan di-cached ketika pengguna mengunjungi halaman tersebut, Jika attribute manifest tidak ditentukan, halaman tidak akan bisa di-cached (tanpa halaman yang ditentukan secara langsung dalam manifest file).

Ekstensi file yang direkomendasikan untuk file manifest adalah: ".appcache"

Sebuah file manifest perlu disajikan dengan tipe media yang benar, yang mana adalah "text/cache-manifest". Harus dikonfigurasikan (diatur) di web server.

The Manifest File

File Manifest

File manifest merupakan file teks sederhana, yang mana memberitahukan browser bagaimana untuk me-cache (dan bagaimana untuk jangan pernah me-cache).

File Manifest memiliki 3 bagian:

  • CACHE MANIFEST - File - file didaftarkan dibawah header ini akan di-cache setelah mereka didownload untuk the first time (waktu pertama).
  • NETWORK - File - file didaftarkan dibawah header ini memerlukan (required) sebuah koneksi ke server, dan tidak akan di-cached
  • FALLBACK - File - file didaftarkan dibawah header ini menentukan halaman fallback jika halaman tidak bisa dimasuki atau diakses

CACHE MANIFEST

Baris pertama CACHE MANIFEST diharuskan:

CACHE MANIFEST
/theme.css
/logo.gif
/main.js

File Manifest diatas mendaftarkan 3 tiga sumber: sebuah file CSS, sebuah gambar GIF, dan sebuah file JavaScript. Ketika file manifest dimuat, browser akan mengunduh tiga file dari root directory dari website. Lalu, kapanpun pengguna sedahg tidak terhubung ke internet, resource masih akan tersedia.

NETWORK

Bagian NETWORK dibawah menentukan bahwa file "login.asp" tidak harus di-cached, dan tidak akan disediakan (secara) offline:

NETWORK:
login.asp

Sebuah tanda bintang bisa digunakan untuk mengindikasikan bahwa semua resource/file lainnya memerlukan sebuah koneksi internet:

NETWORK:
*

FALLBACK

Bagian FALLBACK dibawah ini menentukan bahwa "offline.html" akan disajikan dalam tempat dari semua file di katalog /html/, pada kasus suatu koneksi internet tidak bisa dibentuk:

FALLBACK:
/html/ /offline.html
Catatan: URI pertama adalah resource (sumber), yang kedua adalah fallback.

Updating the Cache

Pembaharuan Cache

Sekali aplikasi di-cache, applikasi tetap di-cached sampai salah satu hal berikut terjadi:

  • User membersihkan browser's cache
  • File manifest dimodifikasi (lihat tips dibawah)
  • Application cache diperbaharui secara otonatis

Example - Complete Cache Manifest File

Contoh - Cache Manifest File Lengkap

CACHE MANIFEST
# 2012-02-21 v1.0.0
/theme.css
/logo.gif
/main.js

NETWORK:
login.asp

FALLBACK:
/html/ /offline.html
Tip: Baris dimulai dengan sebuah tanda "#" merupakan baris comment, tapi bisa juga melayani maksud lainnya. Sebuah cache applikasi hanya diperbaharui ketika file manifestnya berubah. Jika kamu mengedit sebuah gambar atau mengubah sebuah fungsi JavaScript, perubahan - perubahan itu tidak akan di-"re-cache". Pembaharuan tanggal (date) dan versi dalam sebuah baris comment merupakan satu cara untuk membuat browser me-"re-cache" file kamu.

Notes on Application Cache

Catatan - catatan di Application Cache

Berhati - hatilah dengan apa yang kamu cache.

Sekali sebuah file di-cache, browser akan meneruskan atau terus menampilkan veris yang di-cache, bahkan jika kamu mengubah file pada server. Untuk memastikan browser memperbaharui cache, kamu perlu mengubah file manifest.

Catatan: Browser mungkin memiliki batas ukuran yang berbeda - beda untuk cached data (beberapa browser memiliki sebuah batas 5MB limit per site).
