What is SVG?

Apa itu SVG ?

  • SVG didirikan untuk Scalable Vector Graphics
  • SVG digunakan untuk mendefinisikan vector-based graphics untuk web
  • SVG mendefinisikan grafik dalam formst XML
  • Grafik SVG TIDAK menghilangkan (lose) kualitas apapun jika grafik di-zoom dan di-resize
  • Setiap element dan setiap attribute dalam file SVG bisa dianimasikan
  • SVG merupakan sebuah rekomendasi W3C
  • SVG terintegrasi (terpadu) dengan standar W3C lain seperti DOM dan XSL

SVG is a W3C Recommendation

SVG merupakan Rekomendasi W3C

SVG 1.0 menjadi rekomendasi W3CZ pada 4 September 2001.

SVG 1.1 menjadi rekomendasi W3C pada 14 Januari 2003.

SVG 1.1 (edisi kedua) menjadi rekomendasi W3C pada 16 Agustus 2011.

SVG Advantages

Keuntungan - keuntungan SVG

Keuntungan dari penggunaan SVG melebihi format image lain (seperti JPEG dan GIF) adalah:

  • SVG image bisa dibuat dan diedit dengan text editor apapun
  • SVG image bisa dicari, di-index, di-script dan diringkas (compressed)
  • SVG image dapat di-skala
  • SVG image bisa dicetak dengan kualitas tinggu di resolusi apapun
  • SVG image dapat di-zoom (dan image bisa di-zoom tanpa degradasi atau penurunan (kualitas))
  • SVG merupakan standar terbuka
  • File SVG adalah XML murni
  • Main competitor (Saingan utama) untuk SVG adalah Flash.

Keuntungan terbesar yang SVG miliki melebihi Flash rela dengan standar lain (contoh XSL dan DOM). Flash mempercayakan pada teknokogi pemilik yang tidak merupakan open source.

Creating SVG Images

Pembuatan SVG Images

SVG image bisa dibuat dengan text editor apapun, tapi ia sering kali tidak menyusahkan (convenient) untuk membuat SVG image dengan program gambar. Seperti Inkscape.
