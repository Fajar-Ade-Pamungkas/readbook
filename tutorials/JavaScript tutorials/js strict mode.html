JavaScript Use Strict

use strict; mendefinisikan bahwa kode JavaScript harus dieksekusi di "strict mode".

The "use strict" Directive

use strict merupakan baru pada JavaScript 1.8.5 (ECMAScript versi 5).

Ini bukan merupakan sebuah pernyataan, tapi sebuah ekspresi tetap (literal expression), Tapi diabaikna oleh JavaScript versi terdahulu.

Tujuan dari use strict adalah untuk mengindikasikan bahwa kode harus dieksekusi di "strict mode" (mode teliti).
Dengan mode teliti, kamu tidak bisa, untuk contoh, menggunakan variabel - variabel tanpa dideklarasikan.
Semua browser modern mendujung use strict kecuali Internet Explorer 9 dan versi sebelumnya:

BROWSER SUPPORT
Direktive
use strict 13.0 10.0 4.0 6.0 12.1

Nomor - nomor pada tabel menentukan browser pertama yang mendukung secara penuh perintah (directive).

Kamu bisa menggunakan strict mode di semua program kamu. itu membantu kamu untuk menulis kode pembersih (cleaner code), seperti mencegah kamu dari penggunaan variabel tanpa deklarasi.
"use strict" juga merupakan string, jadi IE 9 tidak akan melempaarkan sebuah error bahkan jika IE 9 tidak mengerti strict mode.

Pendeklarasian strict mode

Strict mode dideklarasikan dengan penambahan tanda kutip dan titik koma menjadi "strict mode"; untuk permulaan dari sebuah naskah (script) atau fungsi JavaScript.

Mendeklarasikan di permulaan script JavaScript, membuatnya memiliki global scope (semua kode akan mengeksekusi pada strict mode).

Contoh

input

"use strict";
var x = 3.14; // ini akan menyebabkan sebuah error sebab x tidak dideklarasikan

output

Contoh

input

"use strict";
myFunction();

function myFunction() {
var y = 3.14; // ini akan menyebabkan sebuah error sebab y tidak dideklarasikan
}

output

"strict mode" dideklarasikan didalam sebuah function, membuatnya memiliki cakupan lokal (hanya kode dalam fungsi yang beda di strict mode):

Contoh

input

var x = 3.14;
// ini tidak akan menyebabkan sebuah error
myFunction();

function myFunction() {
"use strict";
var y = 3.14; // ini akan menyebabkan sebuah error
}

output

Sintaksis "use strict";

Penulisan, untuk pendeklarasian strict mode, didesign untuk kecocokan dengan versi JavaScript terdahulu.
Penyusunan sebuah numeric literal (4 + 5;) atau sebuah string literal ("john Doe") pada sebuah program JavaScript tidak memiliki efek samping. Secara sederhana menyusun bukaj untuk variabel yang ada dan mati.
Jadi "use strict"; hanya hal yang penting untuk penyusun - penyusun baru yang "mengerti" arti dari ini.

Kenapa Mode Strict ?

Strict mode mambuatnya lebih mudah untuk menulis "secure" (keamanan) JavaScript.
Strict mode mengubah dulu penerimaan "sintaksis buruk" masuk ke error yang sesungguhnya.
Sebagai contoh, pada JavaScript normal, kesalahan pengetikan nama variabel menciptakan sebuah variabel global. Pada mode strict, ini akan mengeluarkan sebuah error, pembuatan itu tidak mungkin secara kebetulan menciptakan sebuah variabel global.
Pada JavaScript normal, Seorang pengembang tidak akan menerima nilai - nilai penugasan feedback error apapun (error feedback assigning values) untuk properti yang tidak bisa dibaca.
Pada mode strict, assignment apapun menjadi sebuah properti yang tidak dapat dibaca, hanya sebuah properti penyerapan, sebuah properti yang tidak ada, sebuah variabel yang tidak ada atau object yang tidak ada, akan melemparkan sebuah error.

Yang Tidak Diizinkan Dalam Mode Strict

Penggunaan variabel tanpa pendeklarasiannya (kata kunci var) tidak diperbolehkan:

Contoh

input

"use strict";
x = 3.14; // ini akan menyebabkan sebuah error

output

Object merupakan variabel juga

Penggunaan object tanpa pendeklarasiannya (kata kunci var) tidak diperbolehkan:

Contoh

input

"use strict";
x = {p1 : 10, p2 : 20 }; // ini akan menyebabkan sebuah error

output

Penghapusan variabel (atau object) tidak diperbolehkan:

Contoh

input

"use strict";
var x = 3.14;
delete x ; // ini akan menyebabkan sebuah error

output

Penghapusan sebuah fungsi tidak diperbolehkan:

Contoh

input

"use strict";
function x(pi1, pi2) {}; delete x(p1, p2) ; // ini akan menyebabkan sebuah error

output

Penyalinan sebuah nama parameter tidak diperbolehkan:

Contoh

input

"use strict";
function x(p1, p2); // ini akan menyebabkan sebuah error

output

Nilai tetap numeric oktal (Octal Numeric Literals) tidak diperbolehkan:

Contoh

input

"use strict";
var x = 010; // ini akan menyebabkan sebuah error

output

Karakter escape octal tidak diperbolehkan:

Contoh

input

"use strict";
var x = "\010"; // ini akan menyebabkan sebuah error

output

Penulisan untuk sebuah properti read-only tidak diperbolehkan:

Contoh

input

"use strict";
var obj = {};
Object.defineProperty(obj, "x", {value:0, writable:false});
obj.x = 3.14; // ini akan menyebabkan sebuah error

output

Penulisan untuk sebuah properti get-only tidak diperbolehkan:

Contoh

input

"use strict";
var obj = {get x() {return 0} };
obj.x = 3.14; // ini akan menyebabkan sebuah error

output

Penghapusan sebuah properti yang tidak bisa dihapus tidak diperbolehkan:

Contoh

input

"use strict";
delete Object.prototype ; // ini akan menyebabkan sebuah error

output

String "eval" tidak bisa digunakan sebagai variabel:

Contoh

input

"use strict";
var eval = 3.14; // ini akan menyebabkan sebuah error

output

String "arguments" tidak bisa digunakan sebagai variabel:

Contoh

input

"use strict";
var arguments = 3.14; // ini akan menyebabkan sebuah error

output

Pernyataan with tidak diperbolehkan:

Contoh

input

"use strict";
with (Math) {x = cos(2)}; // ini akan menyebabkan sebuah error

output

Untuk alasan keamanan, eval() tidak diperbolehkan untuk menciptakan variabel - variabel pada scope dari yang mana ia dipanggil

Contoh

input

"use strict";
eval("var x = 3.14");
alert(x) // ini akan menyebabkan sebuah error

output

Pada function memanggil seperti f(), nilai ini asala objet global. Pada mode strict, ia sekarang tak terdefinisi.

Future Proof!

Kata kunci yang dipesan untuk versi JavaScript masa depan TIDAK bisa digunakan sebagai nama variabel pada mode strict.

Mereka adalah:

  • implements
  • interface
  • let
  • package
  • private
  • protected
  • public
  • static
  • yield

Contoh

input

"use strict";
var public = 300; // ini akan menyebabkan sebuah error

output

PERHATIKANLAH!

Directive "use strict" hanya dianggap di permulaan dari sebuah naskah (script) atau sebuah fungsi (function).
