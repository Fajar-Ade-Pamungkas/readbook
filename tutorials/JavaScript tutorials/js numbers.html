Nomor JavaScript

JavaScript hanya memiliki satu tipe Number.
Number bisa ditulis dengan atau tanpa desimal.

Number JavaScript

Number bisa ditulis dengan atau tanpa desimal.

Contoh

input

var x = 32.00; // dengan decimal

var y = 32; // tanpa desimal

ektra besar atau ektra kecil bisa ditulis dengan ilmu pengetahuan (exponential atau ilmu aljabar).

Contoh

input

var x = 123e5; // 12300000

var y = 123e-5; // 0.00123

JavaScript Numbers are Always 64-bit Floating point

Tidak seperti bahasa pemrograman kebanyakan, JavaScript tidak mendefinisikan perbedaan tipe nomor, seperti integers (bilangan bulat), short, long, floating point, dan lain sebagainya.

Number JavaScript selalu disimpan sebagai doubel precision floating point numbers, Berdasarkan Standar IEEE 754 .

Format ini menyimpan number di 64 bit, yang mana number (fraction/pecahan) disimpan di bit 0 sampai 51, exponent (menyangkut Aljabar) di bit 52 sampai 62dan sign (tanda operator) di bit 63.

Tabel Floating Point
Value (Aka fraction/mantissa) Exponent Sign
52 bit (0 - 51) 11 bit (52 - 62) 1 bit (63)

Precision (Ketelitian)

Integer (number tanpa sebuah periode atau notasi eksponen atau aljabar) adalah dipertimbangkan ketepatan sampai 15 digit.

Contoh Sederhana

input

var x = 999999999999999;   // x akan menjadi 999999999999999
var y = 9999999999999999;  // y akan menjadi 10000000000000000

output

Maksimum nilai aritmatika yaitu 17 digit, tapi floating point arithmetic tidak selalu akurat 100%,

Contoh Sederhana

input

var x = 0.1 + 0.2;   // x akan menjadi 0.30000000000000004

output

Nilai dari 1.0 + 0.2 adalah ....

Untuk mengatasi masalah diatas, gunakan operator * dan /.

Contoh Sederhana

var x = (0.1 * 10 + 0.2 * 10) / 10;   // x akan menjadi 0.3

output

Nilai dari (1.0 * 10 + 0.2 * 10) / 10 adalah ....

Hexadecimal

JavaScript menterjemahkan nilai terap numeric (klasifikasi) sebagai Hexadecimal jika mereka didahului dengan 0x.

Contoh Sederhana

input

var x = 0xFF ;   // x akan menjadi 255

output

Hexadecimal dari 0xFF adalah ....

!

 Jangan pernah gunakan nomor dengan awalan nol (seperti 07).
Beberapa versi JavaScript menterjemahkannya sebagai bilangan octal jika nomor ditulis dengan awalan 0 (nol).

Oleh default (settingan pabrik), JavaScript menampilkan number sebagai dasar 10 bilangan desimal.

Tapi kamu bisa menggunakan metode toString() untuk output number sebagai 16 dasar (hexadecimal), 8 dasar (octal), 2 dasar (binary).

CONTOH Sederhana

input

var myNumber = 128;
myName.toString(16);
myName.toString(8);
myName.toString(2)

output

Nilai 128 menjadi hexadecimal, octal dan binary...

Infinity (Jumlah Tak Terbatas)

Infinity (atau -Infinity) adalah nilai JavaScript akan dikembalikan jika kamu menghitung sebuah number diluar number yang barang kali paling besar:

CONTOH DETAIL

input

!DOCTYPE html

html

head

/head

body

p id="waktu" /p

script

function tampilkan() {
var myNumber = 2;
var teks = "";
while (myNumber != Infinity){
myNumber = myNumber * myNumber;
teks = teks + myNumber + "br"";
}
x=document.getElementById("waktu").innerHTML=teks;

}

/script

/body

/html

output

Tekan tombol dibawah ini untuk menampilkan waktu

Dibagi dengan 0 (nol) juga menjadi Infinity.

Contoh Detail

input

var myNumber = 2 / 0;
// x akan menjadi Infinity
var myNumber = -2 / 0;
// x akan menjadi -Infinity

output

Contoh Detail

typeof Infinity;
// Infinity adalah sebuah Number, itu dan akan mengembalikan menjadi Number

output

NaN (Not a Numbers)

NaN adalah sebuah kata yang dipesan oleh JavaScript yang mengindikasikan sebuah nilai bukan sebuah nomor.

Pencobaan menjumlahkan number dengan sebuah string akan menjadi NaN.

Contoh Sederhana

input

var x = 100 * "jeruk";
// x akan menjadi NaN

output

Bagaimanapun jika, jika string berisi sebuah nilai numeric, maka itu akan menjadi number.

Contoh Sederhana

input

var x = 100 * "90";
// x akan menjadi sebuah number

output

Kamu bisa gunakan fungsi javaScrip global isNaN() untuk mencari apakah sebuah nilai termasuk NaN atau tidak.

Contoh Sederhana

input

var x = 100 * "apel";
isNaN(x);
// mengembalikan TRUE, karena x bukan merupakan tipe Number.

output

Jika kamu menggunakan NaN sebagai operasi matematika, maka itu akan menjadi NaN juga.

Contoh Sederhana

input

var x = NaN;
y = 27;
z = x + y;
// akan mengembalikan NaN

output

Atau akan menghasilkan nilai Gabungan (concatenate)

Contoh Sederhana

input

var x = NaN;
y = "27";
z = x + y;
// akan mengembalikan NaN27

output

NaN adalah Number, Jadi apabila menggunakan kata kunci typeof maka akan mengembalikan Number

Contoh Sederhana

input

var x = NaN;
typeof x;
// akan mengembalikan Number, karena NaN adalah Number

output

Number Bisa Menjadi Object

Secara normal number JavaScript adalah nilai sederhana dari literal: var x=123;

Tapi Number bisa juga didefinisikan sebagai object dengan kata kunci: var y = new Number(123)

Contoh Sederhana

input

var x = 123;
// akan mengembalikan Number
var y = new Number(123);
// akan mengembalikan Object

output

!

 Jangan membuat Number sebagai objecr karena itu dapat memperlambat kecepatan eksekusi.

Kata kunci new memperumit kode. Ini bisa memproduksi hasil yang keliru.

Ketika menggunakan assignment (==) maka kedua variabel tersebut terlihat sama.

Contoh Sederhana

input

var x = 123;
var y = new Number(123);
var z = (x==y);
// mengembalikan TRUE (benar) karena keduanya memiliki nilai yang sama.

output

Ketika menggunakan operator equality ====, kedua variabel tidak sama, karena operator === mengekspektasikan kesamaan keduanga antara nilai dan type.

Contoh Sederhana

input

var x = 123;
var y = new Number(123);
var z = (x===y);
// mengembalikan FALSE (salah) karena keduanya memiliki nilai yang sama namun dengan tipe yang berbeda.

output

Atau lebih buruk, Object tidak bisa dibandingkan antara satu sama lain.

Contoh Sederhana

input

var x = new Number(123);
var y = new Number(123);
var z = (x===y);
// mengembalikan FALSE (salah) karena Object tidak dapat dibandingkan dengan sesama object.

output

!

 Object JavaScript tidak bisa dibandingkan.

Metode dan Properti Number

Primitive values atau nilai sederhana (seperti 3.14 atau 2004) tidak memiliki metode atau properti (karena mereka bukan merupakan sebuah object).

Tapi dengan JavaScript, metode dan properti juga tersedia untuk nilai sederhana, karena JavaScript menganggap nilai sederhana sebagai object ketika pengeksekusian metode dan properti.

Properti Number

Tabel Floating Point
Properti Definisi
MAX_VALUE Mengembalikan nilai terbesar dalam JavaScript
MIN_VALUE Mengembalikan nilai terkecil dalam JavaScript
POSITIVE_VALUE Melambangkan Infinity positif (mengembalikan di overflow)
NEGATIVE_VALUE Menggambarkan Infinity negatid (mengembalikan di overflow)
NaN Menggambarkan sebuah nilai "Not-a-Number" (bukan termasuk tipe Number)

Contoh Sederhana

input

var x = 1234567890;
var y = x.MAX_VALUE;
// mengembalikan nilai terbesar dari variabel x;

output

Properti Number termasuk selubung number object JavaScript yang disebut Number.

Properti ini bisa juga di akses seperti Number.MAX_VALUE.

Penggunaan Number.MAX_VALUE, yang mana myNumber adalah sebuah variabel, ekspresi, atai nilai, akan mengembalikam undefinied

Contoh Sederhana

input

var x = 1234567890;
var y = x.MAX_VALUE;
// Akan menjadi Undefined;

output

!
