JavaSceipt Menggunakan 32 bit Operand Bitwise

JavaScript menyusun nomor - nomor sebagai 64 bit floating point number (nomor dengan nilai titik terapung) yang ditunjukan pada 32 bit nomor binary (biner).
Sebelum sebuah pengoperasian bitwise dilakukan, JavaScript mengubah number menjadi 32 bit signed integer (bilangan bulat yang disetujui).
Setelah pengoperasian bitwise dilakukan, hasil diubah kembalik menjadi 64 bit JavaScript Number (Nomor JavaScript).

Contoh dibawah menggunakan 4 bit nomor biner tidak ditandatangani. Karena itu ~ 5 mengembalikan 10.
Sejak JavaScript menggunakan 32 bits signed integer, ia tidak akan mengembalikan 10, ia akan mengembalikan -6.

00000000000000000000000000000101 (5)
11111111111111111111111111111010 (~5 = -6)

Sebuah signed integer menggunakan bit paling kiri ssbagai minus sign.

Ketika sebuah bitwise AND dilakukan pada sepasang bit, ia mengembalikan 1 jika kedua bit 1.

JavaScript (Zero Fill) Bitwise Left Shift (<<)

Ini adalah sebuah zero fill left shift, satu atau lebih bit 0 didorong masuk dari kanan dan bit paling kiri di *fall off:

Desimal Binary
5 00000000000000000000000000000101
5 << 1 00000000000000000000000000000001 (10)

Bitwise (Zero Fill) Left Shift

Input

var x = 5 << 1;

Output

JavaScript (Sign Preserving) Bitwise Right Shift (>>)

Ini adalah sebuah sign preserving left shift. Tiruan dari bit paling kiri didorong masuk dari kiri dan bit - bit paling kiri di *fall off:

Desimal Binary
-5 11111111111111111111111111111011
-5 >> 1 11111111111111111111111111111011 (-3)

Bitwise (Sign Preserving) Right Shift

Input

var x = -5 >> 1;

Output
