JavaScript Performance

Bagaimana cara untuk meningkatkan kecepatan kode JavaScript Kamu.

Kurangi Aktivitas Dalam Loop

Loop sering digunakan dalam Pemrograman.
Setiap elemen dalam sebuah loop, dimasuk kedalam pernyataan for, lalu dieksekusi untuk setiap perkataan yang diulang - ulang (pengulangan kata) dari loop (putaran).
Mencari pernyataan atau kepemilikan (assignment) bisa ditempatkan diluar loop:

Kode yang buruk

if(i = 0; i < arr.length; i++) {

Kode yang baik

x = arr.length;
if(i = 0; i < x; i++) {

Kode yang buruk mengakses properti length dari sebuah Array setiap waktu loop diulang - ulang.
Kode yang lebih baik mengakses properti length diluar loopdan membuat loop berjalan lebih cepat

Kurangi Akses DOM

Pengaksesan HTML DOM sangat lambat, dibandingkan dengan peryataan JavaScript lainnya.
Jika kamu mengira mengakses sebuah element DOM beberapa waktu, mengaksesnya sekali, dan menggunakannya sebagai variabel lokal.

Contoh

Input

obj = document.getElementById("demo");
obj.innerHTML= "Helo world !";

Output

Kurangi Ukuran DOM

Jagalah Number element - element pada HTML DOM small.
Ini akan selalu meningkatkan halaman saat memuat, dan menyumbang percepatan ( page display) terutama pada perangkat yang lebih kecil.
Setiap percobaan mencari DOM (seperti getElementsByTagName) akan bermanfaat dari perangkat yang lebih kecil.

Hindarilah Variabel yang Tidak Diperlukan

Jangan membuat variabel baru jika kamu tidak merencanakan untuk menyimpan nilai - nilai.
Sering kali kamu bisa mengganti kode seperti ini:

var fullName = firstName + " " + lastName;
document.getElementById("demo").innerHTML = fullName;

Dengan ini:

document.getElementById("demo").innerHTML = firstName + " " + lastName;

Penundaan Memuat JavaScript

Peletakan script kamu dibawah page body, membiarkan web memuat halaman pertama.
Saat JavaScript sedang mengunduh, browser tidak akan memulai apapun unduhan yang lainnya, Pada penambahan aktivitas penguraian (parsing) dan penyumbangan (rendering) mungkin diblok.

!

 Spesifikask HTTP mendefinisikan bahwa browser lebih baik tidak mengunduh lebih dari dua dalam komponen parallel.

Sebuah alternatif untuk menggunakan defer="true" dalam script tag. Attribute defer menentukan naskah (script) yang harus dieksekusi setelah halaman selesai diurai, tapi itu hanya bekerja pada external script.

Jika memungkinkan, kamu bisa menambahkan naskah kamu (script) ke halaman dengan kode, setelah halaman sudah dimuat:

Contoh

script
window.onload = downScripts;

function downScripts() {
    var element = document.createElement("script");
    element.src = "myScript.js";
    document.body.appendChild(element);
}
/script

Hindarilah Penggunaan With

Hindarilah penggunaan kata kunci with, ia memiliki efek buruk terhadap kecepatan, dan juga mengacaukan jangkauan (scope) JavaScript.

Kata kunci with, TIDAK DIPERBOLEHKAN dalam strict mode.
