Point pada chapter ini adalah mengetahui masalah - masalah yang biasa terjadi pada JavaScript.

Secara Tidak Sengaja Menggunakan Operator Assignment

Program JavaScript mungkin mengeluarkan hasil yang tidak benar jika seorang programer secara tidak sengaja menggunakan sebuah operator assigment (=), alternatif dari operator perbandingan (==) pada sebuah pernyataan if.
Pernyataan if ini mengembalikan false (seperti yang dibayangan) karena x tidak sama dengan 10:

Contoh

Input

var x = 0;
if(x == 10)

Output

Apakah nilai x sama dengan 10 ?

Pernyataan if ini mengembalikan true (tidak seperti yang dibayangan) karena x adalah 10:

Contoh

Input

var x = 0;
if(x = 10)

Output

Apakah nilai x sama dengan 10 ?

Pernyataan if ini mengembalikan false (tidak seperti yang dibayangan) karena 0 adalah false:

Contoh

Input

var x = 0;
if(x = 0)

Output

Apakah nilai x sama dengan 0 ?

!

 suatu assigment (simbol kepemilikan) selalu mengembalikan nilai assignment.

Perkiraan Perbandingan Lepas

Pada perbandingan teratur (regular comparison) tipe data bukanlah masalah, pernyataan if ini mengembalikan true:

Contoh

Input

var x = 10;
var y = "10";
if(x == y)

Output

Apakah nilai variabel x sama dengan variabel y ?

Pada perbandingan teliti (strict comparison) tipe data itu penting, pernyataan if ini mengembalikan false:

Contoh

Input

var x = 10;
var y = "10";
if(x === y)

Output

Apakah variabel x dan y memiliki kesamaan nilai dan tipe data?

Ini adalah kesalahan yang biasa terjadi, untuk mengingatkan pernyataan switch menggunakan strict comparison:

Pada kasus ini switch akan menunjukan alert:

Contoh

Input

var x = 10;
switch(x) {
case 10:
alert("Helo guys !");
}

Output

alert akan tampil seperti tanda peringatan, so don't panic !!!

Pada kasus ini switch tidak akan menunjukan alert:

Contoh

Input

var x = 10;
switch(x) {
case "10":
alert("Helo guys !");
}

Output

Alert ini tidak akan tampil !

Kebingungan Addition (Penambahan) dan Concatenation (Penggabungan)

Addition adalah tentang menambahkan Number.
Concatenation adalah tentang penggabungan String.
Dalam JavaScript kedua pengoperasian ini menggunakan operator yang sama yaitu + (tambah).
Karena itu, penambahan antara hasil sesama Number akan berbeda dengan penambahan String:

Contoh

Input

var x = 10 + 5; // menjadi 15
switch y = 10 + "5"; // menjadi 105

Output

Ketika menambahkan dua variabel, ini dapat menyulitkan untuk mengantisipasi hasil:

Contoh

Input

var a = 10 ; var b = 5 ; var c = a + b; // menjadi 15

var x = 10 ; var y = "5" ; var z = x + y; // menjadi 105

Output

Kesalah Pahaman Float

Semua Number dalam JavaScript disusun sebagai 64-bit Floating Point Numbers (float).
Semua bahasa pemrograman, termasuk JavaScript, memiliki kesulitan - kesulitan ketepatan nilai float point (bilangan dengan titik atau bilangan desimal).

Contoh

Input

var a = 0.1 ; var b = 0.2 ; var c = a + b; // nilai dari c bukanlah 0.3

if( c == 0.3); // jika menggukan tes if maka akan gagal

Output

Untuk mengatasi masalah ini, untuk membantu gunakan operator bagi dan kali:

Contoh

Input

var a = 0.1 ; var b = 0.2 ; var c = (a * 10 + b * 10)/ 10; // akan menjadi 0.3

Output

Penjedaan Suatu Pernyataan Return

Ini adalah prilaku JavaScript default untuk menutup pernyataan secara otomatis di akhir baris.

Oleh karena itu, Kedua contoh dibawah mengembalikan hasil yang sama:

Contoh 1

Input

function myFunction(a) {    
var power = 10
    return a * power
}

Output

Contoh 2

Input

function myFunction(a) {    
var power = 10 
    return a * power
}

Output

JavaScript juga mengizinkan kamu untuk menjeda sebuah pernyataan var ( var statement) kedalam dua baris.
Karena itu, contoh 3 juga mengembalikan hasil yang sama:

Contoh 3

Input

function myFunction(a) {    
var
power = 10
    return a * power
}

Output

Tapi apakah yang terjadi jika kamu menjeda pernyataan return kedalam dua baris seperti ini:

Contoh 4

Input

function myFunction(a) {    
var power = 10
    return
a * power
}

Output

JavaScript akan mengembalikan undefinied !
Kenapa? karena JavaScript merasakan apa yang kamu maksud.:

Contoh 5

Input

function myFunction(a) {    
var
power = 10
    return;
a * power
}

Output

Penjelasan

Jika sebuah pernyataan tidak lengkap seperti:

var

JavaScript akan mencoba untuk melengkapi pernyataan tersebut dengan pembacaan pada baris baru:

power = 10;

Tetapi semenjak pernyataan ini diselesaikan

return

JavaScript akan secara otomatis menutupnya seperti

return;

Ini terjadi karena pernyataan (akhiran) penutup dengan semicolon adalah opsi di JavaScript.
JavaScript akan menutup pernyataan return di akhir baris, karena ia adalah sebuah pernyataan pelengkap.

 Jangan pernah menjeda sebuah pernyataan return

Pengaksesan Array dengan Index Nama

Banyak bahasa pemrograman yang mendukung array dengan index nama.
Array dengan index nama disebut dengan Associative Array (hashes).
JavaScript TIDAK mendukung areay dengan index nama.
Pada JavaScript Array bisa diakses dengan index nomor:

Contoh

Input

var player = [];
player[0] = "Draxler";
player[1] = "De Bruyne";
player[2] = "Özil";
var x = player.length; // player.length akan mengembalikan banyak item
var y = player[2]; // player[2] akan mengembalikan item "Özil"

Output

Di JavaScript Object menggunakan index nama.
Jika kamu menggunakan index nama, ketika pengaksesan Array, maka JavaScript akan mendefiniskan ulang array menjadi object standar.
Setelah otomatis pendefinisian ulang metode dan properti array akan mengeluarkan hasil undefinied (yang tidak jelas) atau hasil yang salah:

Contoh

Input

var player = [];
player["asRoma"] = "Draxler";
player["manCty"] = "De Bruyne";
player["arsenal"] = "Özil";
var x = player.length; // player.length akan mengembalikan 0
var y = player[2]; // player[2] akan mengembalikan undefined

Output

Mengakhiri sebuah Pendefinisian Array Dengan Koma

Salah

points = [40, 100, 1, 5, 25, 10,];

Beberapa mesin JSON dan JavaScript akan gagal, atau berjalan dengan tidak terduga.

Benar

points = [40, 100, 1, 5, 25, 10];

Mengakhiri sebuah Pendefinisian Object Dengan Koma

Salah

person = {firstName:"John", lastName:"Doe", age:46,}

Beberapa mesin JSON dan JavaScript akan gagal, atau berjalan dengan tidak terduga.

Benar

person = {firstName:"John", lastName:"Doe", age:46}

Undefinied Bukan Null

Dengan JavaScript Null untuk object, undefinied untuk variabel, properti dan metode
Untuk (menjadi) Null, object harus didefinisikan atau dijelas, jika tida akan menjadi undefinied.
Jika kamu mau mengetes jika sebuah object hidup, ini akan melemparkan error jika object adalah undefinied:

Salah

if(myObj !== Null && typeof myObj !== "Undefinied")

Oleh karena itu, kamu harus mengetes typeof() terlebih dahulu:

Benar

if(typeof myObj !== "Undefinied" && myObj !== Null)

Perkiraan Jangkauan Blok Kode (Code Block Scope)

JavaScript TIDAK membuat jangkauan baru untuk masing - masing blok kode.
Itu adalah benar dalam kebanyakan bahasa pemrograman, tapi TIDAK BENAR di JavaScript.
Ini adalah kesalahan umum, Diantara para pengembang JavaScript, untuk mempercayai bahwa kode ini mengembalikan undefinied.

Contoh

for (var i = 0; i <10; i++) {
    // some code
}
return i;
