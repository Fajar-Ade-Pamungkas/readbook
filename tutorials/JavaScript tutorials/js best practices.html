Latihan Terbaik JavaScript

Hindarilah variabel global, hindari new, hindari ==, hindari eval().

Hindarilah Variabel Global

Minimalkan menggunakan variabel global.
Termasuk semua type data, fungsi dan object.
Variabel global dan fungsi bisa ditulis lebih oleh berkas yang lainnya.
gunakanlah alternatif variabel lokal, dan belajar bagaimana menggunakan closures

Selalu Deklarasikan Variable Lokal

semua variabel digunakan dalam fungsi harus dideklarasikan sebagai lokal variabel.
semua lokal variabel harus dideklarasikan dengan kata kunci var, jika tidak mereka akan menjadi variabel global.

!

 strict mode tidak memperbolehkan variabel tanpa deklarasi

Pendeklarasian di Atas

Ini adalah latihan persandian yang baik untuk medeklarasikan semuanya di atas di masing - masing naskah (script) atau (function).

Saran seperti:

  • Berilah cleaner code (langkah satu baris)
  • sediakan satu tempat untuk melihat variabel lokal
  • Membuatnya lebih mudah menghindari variabel global yang tak dikehendaki (menyataakan tak secara langsung)
  • Kurangilah kemungkinan pendeklarasian yang tak dikehendaki.
// Declare at the beginning
var firstName, lastName, price, discount, fullPrice;

// Use later
firstName = "John";
lastName = "Doe";

price = 19.90;
discount = 0.10;

fullPrice = price * 100 / discount;

Ini juga saran untuk loop for:

// Declare at the beginning
var i;

// Use later
for (i = 0; i < 5; i++) {

!

 Dengan default, JavaScript menggerakan semua deklarasi ke atas (JavaScript Hoisting).

Initialize Variabel

Latihan pengkodean yang baik untuk initialize variables ketika kamu mendekalariskannya.

Saran seperti:

  • Berilah cleaner code
  • Sediakanlah a single place (satu tempat) untuk initialize variables.
  • Hindarilah nilai undefinied
// Declare and initiate at the beginning
var firstName = "",
lastName = "",
price = 0,
discount = 0,
fullPrice = 0,
myArray = [],
myObject = {};

!

 Penginisialam variabel menyediakan sebuah ide yang bermaksud untuk menggunakan (dan intended data type).

Jangan Pernah Deklarasikan Object String, Object Number, dan Object Boolean

Selalu perlakukan String, Number dan Object sebagai nilai sederhana atau nilai asli (primitive values). Bukan sebagai Object.
Pendeklarasian tipe - tipe ini sebagai Object, dapat memperlambat kecepatan pengeksekusian JaavaScript dan menghasilkan efek samping yang buruk.

Contoh

Input

var x = "John";         
var y = new String("John");
(x === y) // mengembalikan false (salah) karena x adalah sebuah String dan y adalah sebuah Object.

Output

Apakah x dan y sama ?

Atau bahkan lebih buruk:

Contoh

Input

var x = new string("John");     
var y = new String("John");
(x === y) // mengembalikan false (salah) karena sesama tipe Object tidak bisa dibandingkan.

Output

Apakah x dan y sama ?

Jangan Gunakan new Object()

  • Gunakanlah {} alternatif dari new Object()
  • Gunakanlah "" alternatif dari new String()
  • Gunakanlah 0 alternatif dari new Number()
  • Gunakanlah false alternatif dari new Boolean()
  • Gunakanlah [] alternatif dari new Array()
  • Gunakanlah {} alternarif dari new Object()
  • Gunakanlah /()/ alternatif dari new RegExp()
  • Gunakanlah function() {} alternatif dari new Function()

Contoh (contoh masih error)

Input

var x1 = {};           // new object
var x2 = "";           // new primitive string
var x3 = 0;            // new primitive number
var x4 = false;        // new primitive boolean
var x5 = [];           // new array object
var x6 = /()/;         // new regexp object
var x7 = function(){}; // new function object

Output

Hati - hati Ketentuan Tipe Otomatis

Berhati - hatilah Number secara tidak sengaja bisa diubah menjadi String atau NaN (Not a Number).
JavaScript dengan bebas tipe, sebuah variabel bisa berisi data tipe yang berbeda, dan sebuah variabel bisa merubah tipe data tersebut.

Contoh

Input

var x = "Hello";           // tipe data x adalah String
x = 10;           // merubah tipe data dari String menjadi Number

Output

Ketika melakukan pengoperasian matematika, JavaScript bisa merubah Number menjadi String.

Contoh

Input

var a = 5 + 7;       // a.valueOf() is 12,  typeof a is a number
var b = 5 + "7";     // b.valueOf() is 57,  typeof b is a string
var c = "5" + 7;     // c.valueOf() is 57,  typeof c is a string
var d = 5 - 7;       // d.valueOf() is -2,  typeof d is a number
var e = 5 - "7";     // e.valueOf() is -2,  typeof e is a number
var f = "5" - 7;     // f.valueOf() is -2,  typeof f is a number
var g = 5 - "x";     // g.valueOf() is NaN, typeof g is a number

Output

Pengurangan String dan String, bukan hal umum tapi bisa mengembalikan NaN (Not a Number)

Contoh

Input

var a = "Hello" - "World !";  // mengembalikan NaN

Output

Gunakan Perbandingan ===

Operator perbandingan = selalu mengubah (untuk tipe yang cocok) sebelum membandingkan.
Operator === memiliki kekuatan membandingkan nilai dan tipe dari variabel.

Contoh

Input

0 == "";        // true
1 == "1";       // true
1 == true;      // true

0 === "";       // false
1 === "1";      // false
1 === true;     // false

Output

Gunakan Paraeter Default

Jika sebuah fungsi dipanggil dengan pernyataan lepas, nilai dari pernyataan tersebut diatur menjadi undefined.
Nilai undefinied bisa menjeda kode kamu, kebiasaan yang benar untuk memberikan nilai default pada pernyataan:

Contoh ( contoh masih error )

Input

function myFunction(x, y) {    
if (y === undefined) {        
y = 0;
    }
}

Output

Bacalah lebih banyak tentang parameter fungsi dan argument di Function Parameters .

Contoh ( contoh masih error )

switch(new Date().getDay() ) {

katakan = "Minggu";
break;
case 1:
katakan = "Senin";
break;

katakan = "Selasa";

katakan = "Rabu";
break;
case 4:
katakan = "Kamis";
break;
case 5:
katakan = "Jum'at";
break;
case 6:
katakan = "Sabtu";
break;
default;
nameDay = "Hari tidak diketahui";
}

Hindarilah Penggunaan eval()

Fungsi eval() digunakan untuk menjalankan teks sebagai kode. Pada hampir semua kasus, ini tidak dibutuhkan untuk menggunakannya.
Karena ini memperbolehkan kode berubah - ubah, dan juga menunjukan sebuah masalah keamanan.
